降权执行的好处：

1、创建普通用户inca，用inca跑Nginx服务，开发运维都使用普通帐号，只要和inca同组，照样可以管理nginx，还解决了root权限太大问题

  3. 使用=和-表示的标题分别表示一级和二级标题
  4. =和-表示标题时，数量是任意数量，没有显示上的区别

  2. 引用可以嵌套，一个 > 表示一层

三、列表

语法

  1. 使用星号（*）、加号（+）、减号（-）表示无序列表
  2. 使用数字加一个英文句号表示有序列表

注意点

  1. 标记后面一定要跟一个空格或者制表符
  2. 有序列表使用的数字不影响最终的输出结果（即使输入的是1. 1. 1.的列表，显示依然是1. 2. 3.)
  3. 列表标记通常在最左边，最多可以缩进3个空格
  4. 列表之间的空行不影响列表的展示
  5. 在列表项目内放入应用，那 >符号就需要缩进
  6. 要避免在行首出现数字+句点+空白，这会被当作列表，可以用\转义句点
  7. 列表第一个数字会被当做列表的起始数字
  8. 在列表项目内放入代码区块的化，该区块就要缩进两次，也就是8个空格或者两个制表符

示例

四、代码

语法

  1. 缩进4个空格或者一个制表符就可以建立代码区块（制表符兼容性差）
  2. 使用3个以上的反引号括起来的内容也是代码区块的内容
  3. 使用一个或者两个以上的反引号括起来的内容是单行代码内容

注意点

  1. 一个代码块会一直持续到没有缩进的那一行或者是文件结尾

六、链接

语法

  1. 行内链接：[展示内容]+(URL链接 “可选title”)

  2. 参考式链接使用：[展示内容]一个可选空格[标识id]

  3. 参考式链接定义：[标识id]+:+一个以上空格或者制表符+URL链接+可选title(可以是单引号、双引号，圆括号括起来)

  4. 自动链接：[展示内容]

注意点

  1. 链接网址也可以用方括号包起来
  2. 可以把title属性放在下一行，也可以加一些缩进，如过网址太长的化，这样会比较好看
  3. 网址定义只有在产生链接的时候用到，并不会直接出现在文件之中
  4. 链接辨别标签（标识id）可以有字母，数字，空白和标点符号，但是并不区别大小写
  5. 隐式链接标记功能让你可以省略指定链接标记，这种情形下，链接标记会视为等同与链接文字，要用隐式链接标记只要在链接文字后面加上一个控的方括号
  6. 链接的定义可以放到文件的任何一个地方
  7. 参考式的链接其实重点不在于它比较好写，而是它比较好读，使用Markdown的参考式链接，可以让文件更像是浏览器最后产生的结果，让你可以把一些标记相关的元数据移到段落文字之外
  8. 多个链接时，先定义的链接有效

example

  1. 众里寻他千百度
  2. 千百度
  3. 快速链接： http://www.baidu.com
  4. 这是一个带有id的参考链接[vue教学例程二][1]
  5. 参考链接定义：[1]: https://space.bilibili.com/503256794

mvn常用命令

发表于 2020-09-09

mvn clean

清理项目生产的临时文件,一般是模块下的target目录

mvn compile

编译源码，编译生成class文件,编译命令，只编译选定的目标，不管之前是否已经编译过，会在你的项目路径下生成一个target目录，在该目录中包含一个classes文件夹，里面全是生成的class文件及字节码文件。

mvn package

项目打包工具,会在模块下的target目录生成jar或war等文件

mvn install

模块安装命令 将打包的的jar/war文件复制到你的本地仓库中,供其他模块使用 -Dmaven.test.skip=true 跳过测试(同时会跳过test compile)

mvn test

测试命令,或执行src/test/java/下junit的测试用例

mvn deploy

发布命令 将打包的文件发布到远程参考,提供其他人员进行下载依赖 ,一般是发布到公司的私服

执行new命令，生成指定名称的文章至博客主目录\source\_posts\postname.md。其中layout是可选参数，默认值为post。有哪些layout呢，请到博客主目录\scaffolds目录下查看，这些文件名称就是layout名称。当然你可以添加自己的layout，也可以编辑现有的layout，比如post的layout默认是博客主目录\scaffolds\post.md

生成postname.md文件后，可使用markdown编辑器输入博客正文

自定义页面

在博客主目录\source\下会生成pagename目录，里面有个index.md，直接编辑就可以了，然后在主题的_config.yml中将其配置显示出来。
上述步骤，也可以手工生成，在博客主目录\source\下手工新建pagename目录，及该目录下的index.md

  • 修改生成的source/tags/categories.md文件,加上type: “categories”
